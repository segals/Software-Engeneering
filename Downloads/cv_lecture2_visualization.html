Detection — Finding Interest Points

We need to find the same point in multiple images, even with different viewpoints or lighting.

  • Corners are good — distinctive in all directions
  • Blobs are also good — regions with distinct structure
  • Flat areas or edges alone are bad — ambiguous position
Description — Feature Vector Around Each Point

For each detected point, compute a compact descriptor: x₁ = [x₁⁽¹⁾, …, x_d⁽¹⁾]

  • Must be distinctive — different things look different
  • Must be invariant — same thing looks the same under transforms
  • SIFT uses gradient histograms for robustness
Matching — Correspondence Between Images

Compare descriptor vectors across images using distance metrics.

  • Nearest neighbor by Euclidean distance
  • Ratio test: d₁/d₂ < threshold (Lowe's trick)
  • Filters out ambiguous matches
Desired Properties key
  • Repeatability — found in multiple views despite transforms
  • Saliency — distinctive description
  • Compactness — far fewer than image pixels
  • Locality — small area, robust to clutter/occlusion
Applications real-world
  • Wide baseline stereo
  • Motion tracking
  • Panorama stitching
  • 3D reconstruction (Photo Tourism)
  • Mobile robot navigation
  • Object/scene recognition
Key Insight: We run detection independently per image — no communication between images. The trick is finding the same points in both despite this constraint.
Interest Point Detection

Good interest points are where the image has strong variation in all directions — corners, blobs.

What Makes a Good Detector?
Click a region type to see why it is or isn't a good interest point.
Flat Region

Shifting a window in any direction → no change in intensity. Cannot determine position.

Edge

Shift along the edge → no change. Only perpendicular shift changes intensity. 1D ambiguity.

Corner ✓

Shift in any direction → large intensity change. Uniquely localized. Great interest point!

Blob ✓

Region with a distinctive structure (dark on bright, or bright on dark). Detected with Laplacian of Gaussian.

Harris Corner Detector

Uses the structure tensor (second-moment matrix) to measure image variation in a window. Live demo below.

How It Works

For a window W around point (x,y), measure the intensity change when shifting by (u,v):

E(u,v) = Σ_{(x,y)∈W} [I(x+u, y+v) − I(x,y)]²

Using Taylor expansion, this becomes the structure tensor M:

M = Σ_{W} [ Ix² IxIy ]
          [ IxIy Iy² ]

The eigenvalues λ₁, λ₂ of M tell us everything:

Eigenvalue space — corner/edge/flat classification
Corner Response Function R formula
R = det(M) − k · trace(M)²
  = λ₁λ₂ − k(λ₁+λ₂)²
  (k ≈ 0.04–0.06)
🔵
R ≫ 0
Corner
λ₁ ≈ λ₂ ≫ 0
🔴
R ≪ 0
Edge
λ₁ ≫ λ₂ ≈ 0
⬜
|R| ≈ 0
Flat
λ₁ ≈ λ₂ ≈ 0

Invariance Properties
  • Rotation invariant — eigenvalues stay the same when rotated
  • Partially illumination invariant — uses only derivatives (shift I→I+b)
  • NOT scale invariant — a corner at one scale is an edge at another!
The Scale Problem
⚠️ A corner at coarse scale may look like an edge at fine scale. Harris corner response depends on window size!

→ We need automatic scale selection.
Automatic Scale Selection

Find interest points as local maxima in position AND scale space simultaneously.

The Key Idea

For each point, compute a scale signature — the filter response as a function of scale σ. The characteristic scale is where this response peaks.

Blue = Image 1 signature · Orange = Image 2 (different scale). Both peak at their characteristic scale.
Laplacian of Gaussian (LoG) filter
LoG = ∂²G/∂x² + ∂²G/∂y²

G(x,y,σ) = (1/2πσ²) exp(-(x²+y²)/2σ²)
  • Blob detector — responds to circular regions
  • Peak response when σ matches blob size
  • Applied at multiple scales σ = σ₀, kσ₀, k²σ₀, …
DoG Approximation efficient
DoG = G(x,y,kσ) − G(x,y,σ)
≈ (k−1)σ² ∇²G
  • Difference of Gaussians ≈ Laplacian of Gaussian
  • Much faster to compute in practice
  • Used in SIFT detector
  • Built from Gaussian pyramid images
Scale-Space Extrema Detection

A keypoint is found where the filter response is a local maximum in 3D space: (x, y, σ)

Build Gaussian pyramid (multiple octaves & scales)
↓
Compute DoG between adjacent scale levels
↓
Find local maxima in 3×3×3 neighborhood (x, y, σ)
↓
Save keypoint: position (x, y) + scale σ
Result: Each detected keypoint has a position AND a scale. This scale is used to normalize the descriptor patch, making it scale-invariant.
SIFT Descriptor [Lowe 2004]

Scale Invariant Feature Transform — extraordinarily robust matching. Can handle viewpoint, illumination, even day vs night.

SIFT Properties
✓ Invariant to:
  • Scale
  • Rotation
  • Illumination shift (I → I+b)
~ Partially invariant to:
  • Camera viewpoint (up to ~60° out of plane)
  • Illumination scale (I → aI)
  • Occlusion, clutter

Feature Matching

Given two sets of SIFT descriptors, find the best correspondences.

Nearest Neighbor Matching

For each descriptor in image 1, find the closest descriptor in image 2 by Euclidean distance:

d(x₁, x₂) = √Σ(x₁ᵢ − x₂ᵢ)²
Gray lines = all matches · Colored lines = after ratio test
Lowe's Ratio Test key trick

Naive nearest neighbor has many false matches. Lowe's ratio test filters them:

d(q, nn₁) / d(q, nn₂) < threshold (e.g. 0.8)

If the nearest neighbor is much closer than the second nearest, the match is confident. If both are similar distance, it's ambiguous — reject it.

✓ Accept (ratio < 0.8)
d₁ = 12.3, d₂ = 45.7
ratio = 0.27 → ACCEPT
✗ Reject (ratio ≥ 0.8)
d₁ = 38.1, d₂ = 41.2
ratio = 0.92 → REJECT
